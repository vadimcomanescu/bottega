# QA evidence lives in a private evidence repository, watched as gif blob pages

QA evidence lived on a permanent `bottega/evidence-<slug>` branch of the project repository (`0003-permanent-evidence-and-reader-contract.md`). The branch failed in practice: every project workflow triggered on push built the evidence branch, and the never-merging branches accumulated in the project repository forever.

The research that settled the replacement (mid-2026, primary sources) found GitHub structurally closed to inline private video: the one surface that renders a video player, comment attachments, rejects token auth at its upload endpoint (cookie-only, HTTP 422; GitHub's own CLI has an open blocked issue on it), and its URLs are anonymized bearer links that anyone holding them can view even on a private repository, which disqualifies them for internal platforms twice over. Raw and blob URLs serve video as downloads. The markdown sanitizer strips video tags. Every commercial agent product either posts screenshots only or links out to its own authenticated viewer; none plays video inside a PR.

Evidence therefore lives in one dedicated private repository per GitHub owner, `<owner>/bottega-evidence`, with no workflows: a push builds nothing, the project repository stays clean, and publishing needs only the git credentials every run environment already has, cloud runners included. Playback uses the one GitHub surface that renders motion behind real repository permissions with zero extra infrastructure: the gif blob page. Close renders each scenario's recording as a gif and links its blob page from the PR body; one click plays the walkthrough in the browser, gated by GitHub's own login. The full recording sits beside it as the fidelity copy, accepted as a download. The costs: gifs are silent and low-frame-rate, and playback is one click away rather than inline; an authenticated viewer service over the same repository was designed and deferred as an optional upgrade, because it adds standing infrastructure for fidelity the gif already covers.

This supersedes the evidence-branch half of ADR 0003; its reader contract is unchanged. Evidence branches that already exist stay, because deleting one still kills the images embedded in a delivered PR.
