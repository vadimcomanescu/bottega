# Prose the owner shaped in conversation is delivered directly

ADR 0001 answered a real failure (hand-built releases skipping the review and the docs sweep) with a rule wider than the failure: every change through a run, no manual path. On 2026-07-22 the over-coverage produced two failures in one day. A run forced a product-launch spec onto a one-file research note (PR #98), and a second run began re-speccing a skills rewrite whose every line the owner had already agreed in conversation, adding an issue, a branch, and a sign-off round to work that was finished except for the typing. The owner stopped it: maestro is not built for developing skills prose.

The decision: the floor is unconditional for every change: a worktree and branch, gates green, one cross-family review of the integrated diff, and a PR the owner merges. Above the floor, the path forks by what the change is. A change to what the software does (hooks, scripts, schemas, tests, workflows) goes through a run, which specs, plans, builds, reviews, and QA-drives it. Prose (skills, references, docs) whose content the owner has already shaped in conversation is authored and delivered directly by the lead session; the conversation is that work's agreement, and a second agreement through the run apparatus adds ceremony and subtracts authoring quality. This amends ADR 0001: its reason survives in the floor; its "no manual path" sentence no longer governs prose.
