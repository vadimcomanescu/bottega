# Evidence branches are permanent, and host-facing text follows a reader contract

QA evidence lives on a `bottega/evidence-<slug>` branch that delivery used to delete after the PR merged, but deleting it kills every embedded raw image retroactively, and even before deletion GitHub's anonymous image proxy cannot authenticate to a private repo, so every screenshot returns 404 for every viewer. We now keep the evidence branch permanently and never delete it, and on a private repo we link each screenshot's blob page instead of embedding it; the cost is one small branch per delivered run, which is cheaper than unreadable evidence.

Every artifact delivery writes for a host reader (the PR body, a followup issue) follows one reader contract: write for a reader who was not in the run and has not read the spec, define or link every non-standard term where it is used, never use a label the document does not itself define, link the record of any prior decision cited, and state what the diff cannot show while cutting what it already shows. The motivating failure was a delivered PR whose labels ("D1".."D4") and terms were defined only inside the same diff or nowhere a reader could reach, and whose deferred findings were recorded as shorthand that pointed at nothing.
