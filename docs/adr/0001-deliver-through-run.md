# Every change to bottega is delivered through a run on bottega itself

Bottega's releases were hand-built by sessions imitating pieces of the pipeline, and because the only doc duty in the system lives inside the run skill's docs sweep, hand-built releases shipped with stale docs: by 0.57.0 the README was missing five of the six entry points. We decided every change to this repo is delivered through /bottega:run on this repo. The run executes the installed version of the method while producing the next, the way a compiler compiles its own successor with the current release. There is no manual path: the run already scales down for trivial work ("trivial work you build yourself; review and QA still happen"), and the review, docs sweep, and QA it never drops are exactly what the hand-built path kept skipping.
